# 1. Record architecture decisions

- Status: accepted
- Date: 2026-06-20

## Context

We need to record the architectural decisions made on this project so future
contributors (human and AI) understand why the system is built the way it is.

## Decision

We will use Architecture Decision Records (ADRs), as described by Michael Nygard.
Each record is a Markdown file in `docs/decisions/`, numbered sequentially.

## Consequences

- Decisions are versioned alongside the code.
- New ADRs follow this format: Context → Decision → Consequences.
- Superseded decisions are kept for history and marked accordingly.
